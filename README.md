# Структура проекта

## Основные компоненты

### 1. Менеджер аккаунтов
- **`manager.py`**: Главный скрипт для управления несколькими аккаунтами. Он читает `accounts.yaml` и запускает парсеры параллельно.
- **`accounts.yaml`**: Файл конфигурации для добавления ваших AdsPower аккаунтов. **Здесь прописываются ID профилей.**

### 2. Парсеры
- **`main.py`**: Парсер для **десктопной** версии Facebook.
- **`mobile_main.py`**: Парсер для **мобильной** версии Facebook.
- **`story_extract.py`**: Логика извлечения данных для десктопа.
- **`mobile_story_extract.py`**: Улучшенная логика извлечения данных для мобильной версии (JS + DOM).

### 3. Вспомогательные модули
- **`config.py`**: Глобальные настройки проекта (таймауты, пути, фильтры).
- **`telegram_client.py`**: Отправка сообщений в Telegram (текст, фото, видео).
- **`media.py`**: Скачивание изображений и видео.
- **`ad_classifier.py`**: AI классификация изображений (определение вертикали).
- **`facebook_links.py`**: Обработка и "разворачивание" ссылок Facebook.
- **`auto_register.py`**: Логика авторегистрации на целевых сайтах.
- **`graphql_parser.py`**: Обработка скрытых данных Facebook (GraphQL).
- **`adspower.py`**: Взаимодействие с API браузера AdsPower.
- **`human.py`**: Эмуляция действий человека (скролл, движения мышью).

---

## Настройка аккаунтов

Все ваши профили AdsPower настраиваются в файле **`accounts.yaml`**.

Пример настройки:
```yaml
accounts:
  - id: "k1951vv8"           # ID профиля в AdsPower
    name: "Facebook Mobile"  # Имя для удобства (любое)
    type: "mobile"           # Тип парсера: "mobile" или "desktop"
    enabled: true            # true = включен, false = выключен

  - id: "k1953u4o"
    name: "Facebook Desktop"
    type: "desktop"
    enabled: true
```

Чтобы добавить новый аккаунт, просто скопируйте блок с `- id` и измените данные.

---

## Запуск

Для запуска всех активных аккаунтов используйте команду:
```bash
python manager.py
```
Менеджер сам запустит нужные версии парсеров (мобильный или десктопный) в зависимости от типа, указанного в `accounts.yaml`.
